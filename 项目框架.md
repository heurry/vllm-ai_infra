修改后项目可以按下面这条主线组织：

```text
PDF / flow.xlsx / XML模板 / 流程图说明
  -> MinerU解析
  -> 语义证据构建
  -> sparse+dense+graph检索
  -> EvidencePack
  -> TemplateFamilyResolver
  -> LLM生成ClassName XML
  -> XML校验/模板族校验/证据一致性校验
  -> 汇总为最终P177__FHC.xml
  -> trace / audit / regression
```

**1. 输入层**

```text
data/processed/mineru/...
  MinerU输出的 md / content_list.json / middle.json / images

pdf-loop/.../flow.xlsx
  11个流程节点，记录 node_name 和 template/class

data/xml_template/
  历史XML模板
  例如:
    ASDM1401_DTC_Clear_Type7.xml
    ADCU1301_DTC_Clear_Type7.xml
    DTC_Clear_Type2.xml
    DTC_Read_Type1.xml
    GEEA30_VMM_Change.xml
    P177__FHC.xml

流程图.md
  人工或OCR整理后的流程图说明
  例如:
    DTC Clear Type7
    DTC Clear Type2
    IO Control Type21
```

**2. 文档解析与证据层**

```text
MinerU outputs
  -> Markdown semantic trunk
  -> content_list provenance
```

生成：

```text
KnowledgeUnit
  用于 sparse/dense 初召回

EvidenceUnit
  用于最终给LLM和trace
  包含:
    section
    paragraph
    table
    table_row
    flowchart_text
    flowchart_image
    reference_text
```

重点变化：

```text
table_row 命中后，要回收 parent table
refer-to 命中后，要回收 flowchart section
```

**3. 检索层**

```text
FlowNode
  -> NodeIntent
  -> QueryPlan
  -> sparse retrieval
  -> Milvus dense retrieval
  -> hybrid merge
  -> Graph RAG
  -> reference expansion
  -> evidence filter
```

输出不再只是若干 candidate，而是：

```text
EvidencePack
```

例如 ASDM：

```text
node: ASDM_DTC_Clear
class_name: ASDM1401_DTC_Clear_Type7
ecu: ASDM1401

parameter_section:
  3:DTC Clear Type7

dtc_table:
  4行完整表格

refer_path:
  ASDM参数文档
    -> Refer to EOL process-DTC Clear Type7
    -> 流程图.md 中 DTC Clear Type7 说明

flowchart_text:
  Req: 14 xx xx xx
  Res: 54
  Retry <= 3
```

**4. 模板族层**

新增离线构建：

```text
scripts/build_template_family_registry.py
```

输入：

```text
data/xml_template/*.xml
流程图.md
```

输出：

```text
data/processed/template_family_registry/template_families.json
```

内容：

```text
TemplateFamily
  family_id
  aliases
  xml_examples
  flowchart_titles
  xml_signature
  required_evidence
  validation_rules
```

例如：

```text
DTC_Clear_Type7
  aliases:
    ASDM1401_DTC_Clear_Type7
    ADCU1301_DTC_Clear_Type7
    DTC Clear Type7(Single DTC NRC Allowed)

  xml_examples:
    ASDM1401_DTC_Clear_Type7.xml
    ADCU1301_DTC_Clear_Type7.xml

  flowchart:
    DTC Clear Type7(Single DTC NRC Allowed)

  signature:
    0x14
    0x54
    ClearDTC
    AllowedNRC
    RetryCount
```

在线：

```text
TemplateFamilyResolver
  FlowNode + EvidencePack
  -> ResolvedTemplateFamily
```

**5. LLM生成层**

旧路线：

```text
LLM生成 ScriptNode Args
```

新路线：

```text
LLM生成当前 ClassName 对应的完整 XML
```

例如：

```text
ASDM_DTC_Clear
  -> ASDM1401_DTC_Clear_Type7.xml
```

输入 prompt 包含：

```text
operation信息
完整参数表
refer-to路径
流程图.md对应说明
同模板族XML示例
XML骨架规范
输出schema
```

输出：

```json
{
  "node_name": "ASDM_DTC_Clear",
  "class_name": "ASDM1401_DTC_Clear_Type7",
  "template_family": "DTC_Clear_Type7",
  "class_xml": "<TaskNode>...</TaskNode>",
  "used_evidence": [],
  "confidence": 0.9,
  "needs_review": false
}
```

输出文件：

```text
data/processed/class_xml/{workload}/{ClassName}.xml
data/processed/class_xml/{workload}/class_xml_plan.json
data/processed/class_xml/{workload}/generation_trace.json
```

**6. 校验层**

三层校验：

```text
GenericTaskNodeValidator
  所有生成XML都要过
  检查 TaskNode / MainNodes / CVNodes / FlowsNodes / XML well-formed

TemplateFamilyValidator
  按 family 检查
  例如 DTC_Clear_Type7 检查:
    ClearDTC数量 == 表格HEX行数
    0x14存在
    0x54存在
    AllowedNRC存在
    RetryCount=3
    RetryCount-1存在

EvidenceConsistencyValidator
  检查 XML 中的业务值来自 EvidencePack
  例如:
    D40292/D40392 必须来自表格HEX
    0x31 必须来自 NRC ALLOWED
    流程标题必须匹配 refer-to
```

**7. 汇总层**

最终汇总：

```text
flow.xlsx
  -> 11个FlowNode
  -> 11个ClassName XML

base workflow:
  data/xml_template/P177__FHC.xml

assembler:
  把生成的ClassName XML与主流程引用关系汇总
  输出最终 P177__FHC.xml
```

也就是说：

```text
P177__FHC.xml 是最终总工作流
ASDM1401_DTC_Clear_Type7.xml 是其中一个 ClassName 的实现
ScriptNode 是主流程中引用 ClassName 的入口
```

**8. Trace与评测层**

输出：

```text

```
