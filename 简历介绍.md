## 二、Agent 架构拆解

可以这样对外讲：

```text
用户/业务输入
  ↓
Diagnostic Agent
  ├── Perception：PDF/流程图解析与结构化入库
  ├── Planner：诊断任务理解与 FlowNode 任务拆解
  ├── Retriever：RAG 检索增强与证据包构建
  ├── Memory：历史模板、案例片段、流程图说明复用
  ├── Tool Executor：Milvus 检索、模板匹配、vLLM 调用、XML 校验
  ├── Generator：基于 vLLM 生成诊断 XML/代码
  ├── Validator：结构校验、模板族校验、证据一致性校验
  └── Observer：trace、audit、regression、生成结果回归分析
```

### 对应关系

| Agent 模块           | 你项目里的实际实现                                                              |
| ------------------ | ---------------------------------------------------------------------- |
| Perception 文档感知    | PDF 转 Markdown、流程图结构化、图片流程图转伪代码/结构化步骤                                  |
| Planner 任务规划       | FlowNode 解析、NodeIntent、QueryPlan、多路 query 生成                           |
| Retriever 检索增强     | sparse / dense / hybrid retrieval、Milvus、Graph RAG、EvidencePack        |
| Memory 记忆复用        | 历史 XML 模板、案例片段、流程图说明、TemplateFamily registry                           |
| Tool Executor 工具调用 | vLLM 服务、RAG 检索工具、模板族匹配工具、XML 校验工具                                      |
| Generator 生成器      | 基于 Prompt 编排和 vLLM 生成诊断 XML/代码                                         |
| Validator 校验器      | TaskNodeValidator、TemplateFamilyValidator、EvidenceConsistencyValidator |
| Observer 可观测       | trace、evidence_chain_report、audit、regression                           |
